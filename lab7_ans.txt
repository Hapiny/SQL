1. В чем различия имени входа (логина) и пользователя?
	логин - понятие доступа на уровне сервера
	пользователь - понятие доступа на уроне базы данных

2. Рассказать о ролях уровня сервера.
	
	Роль – это именованный элемент базы данных, который объединяет несколько привилегий.

	sysadmin - разрешено выполнять любые действия в SQL Server
	setupadmin - управление связанными серверами
	serveradmin - доступно конфигурирование и выключение сервера
	securityadmin - управляет учетными записями и правами на создание базы данных
	processadmin - может управлять процессами запущенными в SQL Server
	dickadmin - управляет файлами SQL Server
	dbcreator - разрешено создавать и модифицировать базы данных
	bulkadmin - может управлять процессом массовой вставки данных	

3. Рассказать о ролях уровня базы данных.
	
	db_owner - члены предопределенной роли базы данных db_owner могут выполнять все действия 
	по настройке и обслуживанию базы данных, а также удалять базу данных в SQL Server.

	db_secutiryadmin - может управлять ролями и разрешениями на уровне базы данных
	db_accessadmin - может удалять или добавлять пользователей базы данных
	db_backupoperator - может выполнять операцию резервного копирования
	db_ddladmin - может выполнять команды DDL
	db_datareader - может просматривать содержимое таблиц базы данных
	db_datawriter - может изменять данные в таблицах
	db_denydatareader - запрещается просматривать данные в любой таблице
	db_denydatawriter - запрещается модифицировать данные в любой таблице
	public - любой пользователь базы данных автоматически становится членом этой роли.

4. Можно ли создать свою роль уровня сервера?
	Нет, и менять предопределенные роли тоже нельзя ( кроме роли publuic)
	Разрешения, назначенные предопределенным ролям сервера (кроме роли public), не могут быть изменены. 
	Начиная с SQL Server 2012, можно создавать пользовательские роли сервера и добавлять разрешения на уровне сервера таким пользовательским ролям.


5. Для чего нужны роли?
	Роль позволяет объединять пользователей выполняющих одинаковые функции для урощения администрирования.

6. Что такое схема?
	Схема - это средство группировки объектов (не пользователей), позволяющее обращаться с набором объектов как с единым целым.


7. Про роль public
	public — эта специальная роль предназначена для предоставления разрешений сразу всем пользователям базы данных.
	Специально сделать пользователя членом этой роли или лишить его членства невозможно. Все пользователи базы данных получают права этой роли автоматически.

8. Как разрешить пользователю предоставлять разрешение другим пользователям?
	При операторе GRANT <OPTION> указывать WITH GRANT OPTION


9. Рассказать про директивы GRANT, DENY и REVOKE.
	GRANT - предоставляет разрешения на защищаемый объект участнику. Общий подход заключается в том, 
	чтобы предоставить <некоторые разрешения> на <некоторый объект> <некоторому пользователю, имени входа или группе> (GRANT…ON…TO).

	DENY - запрещает разрешение для участника.

	REVOKE - удаляет разрешение, выданное или запрещенное ранее.
